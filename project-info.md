# Collaboration

## Workflow

1. Divide a task/feature into sub-tasks and create a github issue for each sub-task.
2. Assign the issue(s) you want to work on to yourself and create a new branch before starting any work.
    - Reference the relevant issue # in each commit message.
3. Assign the merge/pull request to someone else so everyone comes into contact with the entire codebase.

## Other

1. Compile with (at least) the following flags: `-Wall`, `-Werror`, `-Wpedantic`.
2. Add the `.out` extension to executables so they get gitignored.
3. I use `snake_case` for all var/method naming, lmk if you prefer otherwise.
